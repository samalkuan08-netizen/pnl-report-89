# P&L-агент Градекс KZ

Веб-приложение: загрузка выгрузки 1С (Анализ счёта с субконто) → автоматическое
заполнение P&L по шаблону с разнесением по дивизионам **ТШО / КПО / Бетон / АУП**.

## Логика
- Иерархия 1С читается по уровням группировки (счёт → статья → подразделение).
- Доходы и расходы разносятся по листам дивизионов; лист `P&L 2026 all` считается формулами.
- Амортизация и страхование счёта 7210 распределяются по подразделениям; чистый АУП — в админ-блок.
- Депозит (доходы от финансирования) → «Прочий доход» отдельно от субсидий.
- Новая статья ≥ 300 000 ₸ → отдельная строка; ниже порога → «Прочие».

- Расходы на финансирование (счёт 73xx) — отдельной строкой в операционном блоке, по подразделениям. Счёт 7400 (курсовые разницы 7430, обмен валюты 7480 и пр.) — в строку «Прочие расходы». КПН (77xx) пропускается. Статьи без подразделения идут на лист concrete и помечаются флагом.
- Контроль: сумма заполненного отчёта сходится с итогом 1С до копейки.

## Файлы
- `app.py` — интерфейс Streamlit
- `pnl_agent.py` — ядро (парсер 1С + словари сопоставления + заполнение)
- `template.xlsx` — **положите сюда ваш шаблон GRD** (или загружайте через интерфейс)

## Локальный запуск
```bash
pip install -r requirements.txt
streamlit run app.py
```

## Деплой на Streamlit Community Cloud
1. Залейте репозиторий на GitHub (вместе с `template.xlsx`).
2. share.streamlit.io → New app → выберите репозиторий и `app.py`.

## Правка словаря сопоставления
Все соответствия «статья 1С → строка отчёта» — в `pnl_agent.py`
(словари `OPEX_CONCEPT`, `OPEX_LABEL`, `ADMIN_CONCEPT`, `INCOME_MAP`). Меняются без правки логики.
